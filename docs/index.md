# 首页

> 笔记不是堆积信息，而是塑造思考的“第二大脑”。让知识可回溯、可复用、可协作，是写作的终极价值。

---

## 为什么要写

- 澄清：把模糊的念头写出来，才能不断迭代为明确的结论。
- 积累：把一次性的思考固化为可复用的资产，减少重复劳动。
- 连接：在不同主题间建立链接，让知识网络自然生长。
- 复盘：通过笔记追踪决策与结果，沉淀方法论与经验教训。

## 怎么更好地写

- 原子化拆分：一条笔记只解决一个问题，避免“大而全”。
- 结构化表达：背景/问题/推导/结论/参考，尽量完整但不过度追求形式。
- 轻量迭代：先记录，再修正；把“完美”留给发布而不是记录。
- 善用图示：用 Mermaid/思维导图/草图解释复杂关系，让抽象具象化。

## 好的习惯

- 命名统一：使用有语义的文件名，降低未来检索成本。
- 链路稳定：优先使用相对路径与目录索引，避免外链失效。
- 关注上下文：写清“为什么这样做”，让读者（也包括未来的自己）快速进入状态。
- 定期整理：把“收件箱”里的内容按主题沉淀到相应目录，保持体系成长。

## 期望的状态

- 让笔记成为你做事的方法与证据；
- 让写作更像呼吸，而不是任务；
- 让知识在网络中流动，并在需要时被轻松地调用。

> 你不需要等到准备好才开始写——写作本身，就是准备的一部分。
